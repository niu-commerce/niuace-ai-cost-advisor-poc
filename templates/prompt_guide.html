Trade Code Format & AI Prompt Guide
Reference for understanding trade code structure, classification rules, and how to fine-tune the AI prompt.
Trade Code Anatomy
PIL-TYPE-SIZE-ACTIVITY[-LOAD]
PIL Fixed prefix — all piling trade codes start with PIL TYPE Pile material / construction method (GEN, PRC, SPUN, BORED) SIZE Pile cross-section or diameter (GEN for generic, or normalised dimensions) ACTIVITY Work scope for this BQ item — what is actually being done LOAD Test load tonnage — only appended for static/dynamic load tests
TYPE — Pile Material & Method
CodePile TypeDescriptionExample Code
GEN General Site-wide items not tied to a specific pile type — survey, equipment mobilisation, records, obstructions PIL-GEN-GEN-SURVEY
PRC Prestressed Reinforced Concrete Square precast concrete piles; most common in Malaysia residential & commercial. Sizes 125×125 to 300×300 PIL-PRC-150X150-DRIVE
SPUN Spun (Centrifugally Cast) Concrete Hollow cylindrical high-strength concrete piles; used for heavy loads. Sizes typically 300 mm–600 mm dia PIL-SPUN-600MM-SUPPLY
BORED Bored / Drilled Piles Cast-in-place concrete in drilled shafts; used in congested sites or where vibration is restricted PIL-BORED-600MM-DRIVE

LOAD Suffix — Test Tonnage

Appended only for static load test activities (LOAD, LOAD-PRELIM, LOAD-SUBSEQ, LOAD-MAINT). Derived from the actual test load in the BQ description.

SuffixLoadTypical Pile Size
36T36 tonnes125×125 mm PRC piles
45T45 tonnes125×125 mm PRC (higher capacity) or 150×150 mm
50T50 tonnes150×150 mm PRC piles
90T90 tonnes200×200 mm PRC or larger piles

⚠ Do NOT add LOAD suffix just because the pile description mentions "working load", "load bearing" or tonnage capacity — only when the item is actually a load test.

Fine-Tuning Discussion Notes
  • 🎯
    Classification Priority Rules PDA always wins over LOAD when both keywords appear. SUPPLY vs DRIVE is determined by whether the pile has already been delivered ("uplift from stack" implies delivered, so classify DRIVE).
  • ⚠️
    False LOAD Triggers Words like "working load", "load bearing", or pile capacity tonnages in supply/drive items should NOT trigger a LOAD activity. Only classify as LOAD when the item is explicitly a test.
  • 📐
    Size Normalisation Contract BQ text often uses loose formatting: "125mm x 125mm", "125 x 125", "125x125". The AI should normalise all to 125X125 (no spaces, uppercase X). Ensure examples in the prompt cover all common variants.
  • 🔧
    How to Add Rules Without Changing Code Edit prompt_tuning_rules.txt in the project root. Rules appended there are automatically merged into the active prompt at runtime. Use this for audit-derived corrections — no deployment needed.
  • 📊
    Confidence Calibration A confidence below 0.65 (shown in yellow in BQ Analyzer) usually means the description is ambiguous or uses unusual wording. These are good candidates for manual review in the Trade Code Library.
  • 🧪
    Testing a New Rule Add the rule to prompt_tuning_rules.txt, then paste a real BQ description into BQ Analyzer and check the trade code and reasoning. Review the "Active Prompt" tab above to confirm the rule is merged correctly.
